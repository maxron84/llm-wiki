## Kernaussagen

### Karpathys Evolution

Der Artikel ordnet das LLM-Wiki-Konzept in Karpathys Werkzeug-Entwicklung ein:

| Datum | Konzept | Shift |
|---|---|---|
| Feb 2025 | Vibe Coding | Code vergessen, KI akzeptieren |
| Dez 2025 | "Magnitude 9 earthquake" | Bewusstsein für KI-Disruption |
| Jan 2026 | Agentic Engineering | Menschen orchestrieren, <1% Code schreiben |
| **Apr 2026** | **LLM Knowledge Bases** | **Über Code hinaus — Wissen kompilieren** |

Der rote Faden: **Markdown wird zur Programmiersprache der KI-Ära.**

### Anwendungsfälle (7)

Antigravity.codes liefert konkrete Einsatzszenarien:

1. **Competitive Intelligence** — Konkurrenten-Websites, Changelog, Job-Postings → Wettbewerbswiki
2. **Technical Due Diligence** — Repos, Docs, Team-Profile → Architektur- und Schulden-Analyse
3. **Academic Literature Review** — 50–100 Paper → Ergebniszusammenfassung, Methodenvergleich, Forschungslücken
4. **Developer Documentation** — READMEs, ADRs, Postmortems → lebendige Wissensbasis für neue Teammitglieder
5. **Product Research** — Support-Tickets, NPS, Feature-Requests → Schmerzpunkte, Priorisierung
6. **Legal & Compliance** — Regulatorische Dokumente → Anforderungs-Mapping, Lückenerkennung
7. **Personal Learning** — Persönliches Kurrikulum, das genau widerspiegelt, was man studiert hat

### Kompilierungsprofile

Verschiedene Wikis brauchen verschiedene Kompilierungsstrategien:

- **Research**: Behauptungen, Evidenzstärke, Methodik, Zitate, Widersprüche
- **Competitive**: Produkt-Features, Preisgestaltung, Positionierung, Teamgröße, Finanzierung
- **Learning**: Konzepte, Voraussetzungen, Schwierigkeitsstufen, Übungen
- **Decision Log**: Entscheidungen, Alternativen, Begründung, Entscheider, Datum

### Fortgeschrittene Muster

- **Multi-Source-Ingestion** — Automatisierte Pipelines: Web Clipper, PDF→Markdown, YouTube-Transcripts, GitHub-Repos, RSS-Feeds
- **Geplante Health-Checks** — Täglich: Neue Dateien prüfen; Wöchentlich: Vollständige Konsistenzprüfung; Monatlich: Web-Verifikation veralteter Fakten
- **Cross-Wiki-Verlinkung** — Verbindungen zwischen mehreren Wissensbases (z.B. ML-Forschung ↔ Produkt-Strategie)
- **CI/CD für Wissen** — Git + automatisierte Health-Checks bei jedem Commit

### Neue Entwicklerrolle

> "The developers who thrive in the next phase won't just be the best prompters or the best agent orchestrators. They'll be the ones with the best knowledge systems."
